# LoyaltyLoops B2B CRM

Бизнес-процессы и техническая реализация B2B CRM: иерархия менеджеров платформы, продажа и жизненный цикл подписок, заявки, блокировки.

## 1. Роли и Доступы (RBAC)

*   **PLATFORM_SUPER_ADMIN (Владелец Платформы)**
    *   Полный доступ ко всем функциям.
    *   Может приглашать и увольнять `PLATFORM_SUPER_MANAGER` и `PLATFORM_MANAGER`.
    *   Может удалять/блокировать кого угодно (кроме самого себя).
    *   Видит полную статистику, все заявки и системные логи.

*   **PLATFORM_SUPER_MANAGER (Старший Менеджер)**
    *   Может приглашать и увольнять только `PLATFORM_MANAGER`.
    *   **Не может** удалить `PLATFORM_SUPER_ADMIN` или другого `PLATFORM_SUPER_MANAGER`.
    *   Может одобрять/отклонять заявки менеджеров.
    *   Видит статистику по всем менеджерам.

*   **PLATFORM_MANAGER (Менеджер)**
    *   Работает «в поле» с партнерами.
    *   Создает заявки: подключение точки (активация подписки), продление, блокировка/разблокировка партнера.
    *   Не может одобрять свои заявки (требуется аппрув сверху).
    *   Видит только свои заявки; нет доступа к системным логам.

## 2. Схема Базы Данных

| Таблица | Назначение | Ключевые поля |
|---|---|---|
| `SystemStaffTable` | Связь пользователя с ролью на платформе | `userId`, `role` |
| `PlatformRequestsTable` | Заявки, требующие одобрения | `type`, `status (PENDING/APPROVED/REJECTED)`, `requesterId`, `approverId`, `targetPoint`, `amount`, `duration`, `isTrial`, `blockReason` |
| `PlatformSubscriptionsTable` | Активные/истекшие подписки | `partnerId`, `tradingPoint`, `startDate`, `endDate`, `amount`, `isTrial`, `isActive` |
| `PlatformInvitesTable` | Коды приглашений в команду платформы | `code`, `role`, `createdBy`, `createdAt` |

## 3. Жизненный цикл Заявки (Request Flow)

Все ключевые действия (активация, продление, блокировка) проходят через систему заявок (`PlatformRequests`).

1.  **Создание (PENDING)**: Менеджер создает заявку.
    *   Указывает `Point ID` (система сама находит Партнера).
    *   Выбирает Тип (Активация, Блок и т.д.).
    *   Указывает условия: Сумма, Длительность, или `TRIAL` (14 дней).
    *   *Важно:* Если партнер находится в статусе `PENDING` (на проверке), менеджер может создать заявку, но одобрить её нельзя до активации партнера.

2.  **Рассмотрение**: Супер-админ или Супер-менеджер видят заявку в списке, детали и профиль партнера.

3.  **Решение**:
    *   **APPROVE**: создается запись в `PlatformSubscriptionsTable`; партнер активируется (если был не активен); точка активируется (`isActive = true`); логируется системное событие.
    *   **REJECT**: заявка переходит в `REJECTED` с указанием причины.

## 4. Автоматизация и Подписки (Subscription Lifecycle)

Фоновый Job проверяет подписки раз в сутки.

1.  **За 3 дня (Warning)**:
    *   Владельцу партнера — **Email** с названием точки.
    *   Менеджерам (инициатор + все менеджеры партнера) — **Email** с названием точки и партнера.
    *   Супер-админам/супер-менеджерам — сводный отчет на почту.
    *   Лог в системный аудит (`INFO`); в кабинете владельца — **желтый Alert**.

2.  **За 1 день (Critical)**:
    *   Владельцу партнера — **Email + SMS**.
    *   Менеджерам — только Email.
    *   Админам — повторный сводный отчет, если не решено.
    *   Лог (`CRITICAL`); в кабинете — **красный Alert «СРОЧНО»**.

3.  **Истечение (Expired)**:
    *   Подписка помечается `isActive = false`.
    *   **Конкретная точка** деактивируется (`isActive = false`, `isTemporarilyPaused = true`); остальные точки партнера работают, если у них свои подписки.
    *   Лог о деактивации.

### Trial (Пробный период)
*   Длительность жестко фиксирована: **14 дней**.
*   Триал выдается **на точку**: повторный триал для точки запрещен (`TRIAL_ALREADY_USED`), учитываются и уже использованные триалы, и pending-заявки с триалом.

## 5. Блокировка Партнера

1.  **Soft Ban** (`BLOCK_PARTNER` → статус `BLOCKED`):
    *   Точки **не** деактивируются в базе (подписка продолжает тикать).
    *   Транзакции (`ScanQr`) блокируются на уровне API с ошибкой `PARTNER_BLOCKED`.
    *   Точки скрываются из публичного поиска и карты (фильтр по `PartnerStatus.ACTIVE`).
2.  **Разблокировка** (`UNBLOCK_PARTNER` → `ACTIVE`): точки мгновенно доступны (при активной подписке).

## 6. API (`/platform`, см. openapi/documentation.yaml)

| Endpoint | Метод | Назначение |
|---|---|---|
| `/platform/join` | POST | Применить код приглашения и получить роль |
| `/platform/invite` | POST | Сгенерировать код приглашения (владелец: SUPER_MANAGER и MANAGER; руководитель: только MANAGER) |
| `/platform/requests` | POST / GET | Создать заявку (менеджер) / список заявок с фильтрами |
| `/platform/requests/{id}/approve` | POST | Одобрить (руководитель+) |
| `/platform/requests/{id}/reject` | POST | Отклонить с причиной (руководитель+) |
| `/platform/alerts` | GET | Алерты по истекающим подпискам |
| `/platform/staff` | GET / DELETE `{id}` | Список сотрудников платформы / увольнение |

## 7. Системные События (Audit Logs)

Логируются в `SystemEventsTable`, доступны в разделе «Аудит»: вход/регистрация, создание заявки, одобрение/отклонение, блокировка/разблокировка партнера, автоматические уведомления, автодеактивация точки.

## 8. Локализация

*   Ключевые языки (**RU, EN, KY, KK, UZ, BE**) — вручную через `web-admin/src/i18n/locales/`.
*   Остальные — автоперевод с защитой бренда «LoyaltyLoop».
*   Все новые ключи B2B CRM синхронизируются во всех языковых файлах.
