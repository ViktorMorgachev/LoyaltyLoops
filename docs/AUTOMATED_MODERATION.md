# Система Социального Рейтинга и Обратной Связи (Social Rating System)

Вместо жесткой системы блокировок мы внедряем систему **Социального Рейтинга**. 
Это позволяет мягко регулировать поведение клиентов и собирать ценную аналитику для бизнеса.

Система работает в двух направлениях:
1.  **Кассир оценивает Клиента** (Internal Trust Score) — защита бизнеса.
2.  **Клиент оценивает Точку** (Service Review) — контроль качества сервиса.

---

## 1. Рейтинг Клиента (Internal Trust Score)

Скрытый рейтинг надежности клиента, который видят только сотрудники.

### Сценарий: Оценка после визита
1.  Кассир завершает транзакцию.
2.  Появляется окно оценки (Звезды 1-5 + Теги).
3.  Оценка влияет на репутацию клиента.

### Алгоритм Расчета: "Последние 20 визитов"
Почему не "за месяц"? Потому что время не лечит характер. Репутация исправляется только новыми хорошими действиями.

*   **Новый клиент**: По умолчанию **4.0** (Стандартный).
*   **Окно**: Среднее арифметическое последних **20 оценок** (скользящее окно).
*   **Формула с учетом Тегов**:
    $$ Rating = \frac{\sum (Stars + TagPenalty)}{N} $$

#### Вес Тегов (Tag Penalty)
Теги влияют на рейтинг сильнее, чем просто звезды.

| Тег (Причина) | Влияние | Логика |
| :--- | :--- | :--- |
| **Нет тега** | Stars (как есть) | Обычное взаимодействие. |
| 🤬 **Агрессия / Хамство** | `Stars - 2` балла | Грубость недопустима. Даже если кассир поставил "3", система посчитает это как "1". |
| 💸 **Неплатежеспособность** | `Stars - 1` балл | Проблемы с картой, отказ от оплаты. |
| 🍵 **Чаевые / Позитив** | `Stars + 0.5` балла | Бонус за лояльность (может поднять рейтинг выше 5.0, компенсируя прошлые грехи). |

### Результат: "Светофор" для Кассира
При сканировании QR-кода сервер возвращает `riskLevel` и цвет:

*   🟢 **ЗЕЛЕНЫЙ (4.5 - 5.0+)** — "Надежный клиент".
    *   *Действие*: Приветствуем, предлагаем новинки.
*   🟡 **ЖЕЛТЫЙ (3.5 - 4.4)** — "Обычный клиент".
    *   *Действие*: Стандартное обслуживание.
*   🟠 **ОРАНЖЕВЫЙ (2.0 - 3.4)** — "Сложный клиент".
    *   *Действие*: Быть внимательнее, четко проговаривать заказ, не спорить.
*   🔴 **КРАСНЫЙ (< 2.0)** — "Вредный клиент".
    *   *Действие*: Максимальная осторожность.

## 2. Защита от "Мести Кассира" (Anti-Abuse)

Механизм защиты клиентов от предвзятого отношения персонала (например, личный конфликт).

1.  **Лимит частоты**:
    *   Учитывается только **одна оценка** от одного кассира одному клиенту в **сутки**.
    *   Это предотвращает "спам единицами".

2.  **Детекция Аномалий (Выбросов)**:
    *   Если один кассир ставит "1 звезду" клиенту, у которого в других 10 магазинах сети рейтинг "5.0" — система **игнорирует** эту единицу (считает её выбросом).
    *   **Алерт Владельцу**: Отправляется уведомление "Ваш кассир [Имя] занижает рейтинг хорошему клиенту [Имя]".

## 3. Рейтинг Торговой Точки (Service Review)

Обратная связь от клиента бизнесу. Позволяет владельцу выявлять проблемы в конкретных филиалах.

### Сценарий: Оценка в приложении
1.  Клиент при получении в райнтайме информации о транзакции можен оценить торговую точку (в будущем планируется оценить в списке транзакций)
2.  Нажимает "Оценить визит":
    *   **Звезды**: 1-5.
    *   **Теги**: "Грязно", "Долго", "Вкусно", "Быстро", "Приветливый персонал".
    *   **Комментарий**: Текстовое поле.

### Админ-панель Владельца: UX/UI
Владелец видит раздел **"Отзывы и Аналитика"**:

1.  **Дашборд (Сводка)**:
    *   **NPS (Net Promoter Score)**: Большой индикатор общего индекса лояльности по всей сети (например, "+45" на зеленом фоне).
    *   **Средний Рейтинг**: 4.8 ⭐ (с динамикой "⬆️ +0.1 за неделю").

2.  **Детальный Список Отзывов**:
    *   Карточки отзывов с фильтрацией по точкам, дате и рейтингу.
    *   *Пример карточки*:
        > **Иван И.** ⭐⭐⭐⭐ (ЦУМ)
        > [Теги: Вкусно, Долго]
        > "Все понравилось, но кофе ждал 15 минут."
        > *Кнопка "Ответить" (В будущем)*

3.  **Тепловая Карта Проблем (Analytics)**:
    *   Визуализация "Болевых точек" бизнеса.
    *   *UI*: Матрица или гистограмма, где по оси X — Торговые Точки, по оси Y — Проблемные Теги.
    *   *Пример*:
        *   🔴 **"Грязно"**: Точка "Азия Молл" (15% отзывов). -> *Инсайт для Владельца: "Срочно проверить клининг в Азия Молл".*
        *   🟡 **"Медленно"**: Точка "Центр" (40% отзывов). -> *Инсайт: "Не хватает бариста в час-пик".*
        *   🟢 **"Вкусно"**: Точка "Юг" (90% отзывов). -> *Инсайт: "Премировать поваров".*

## 4. Техническая Реализация (БД)

### 1. Таблица `ClientRatings` (Оценки клиентов)
Хранит историю оценок поведения клиента.

```sql
CREATE TABLE client_ratings (
    id UUID PRIMARY KEY,
    partner_id UUID NOT NULL,
    trading_point_id UUID NOT NULL,
    cashier_id UUID NOT NULL,
    user_id UUID NOT NULL,
    rating INT NOT NULL,            -- 1-5
    tags TEXT,                      -- "AGGRESSION,TIP" (Enum)
    is_ignored BOOLEAN DEFAULT FALSE, -- Флаг Anti-Abuse (выброс)
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2. Таблица `ServiceReviews` (Оценки сервиса)
Хранит отзывы клиентов о точках.

```sql
CREATE TABLE service_reviews (
    id UUID PRIMARY KEY,
    partner_id UUID NOT NULL,
    trading_point_id UUID NOT NULL,
    user_id UUID NOT NULL,
    rating INT NOT NULL,            -- 1-5
    tags TEXT,                      -- "SLOW,DIRTY"
    comment TEXT,
    is_read_by_owner BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3. Таблица `LoyaltyCard` (Кэш)
```sql
ALTER TABLE loyalty_cards ADD COLUMN trust_score FLOAT DEFAULT 4.0; -- Текущий рейтинг (кэш)
ALTER TABLE loyalty_cards ADD COLUMN fraud_flag BOOLEAN DEFAULT FALSE; -- Черная метка
```

## 5. Конфигурация
Пороги рейтинга заложены в системе, но могут быть переопределены:
*   `RATING_WINDOW_SIZE`: 20 (визитов)
*   `ANTI_ABUSE_SIGMA`: 2.0 (отклонение от среднего для детекции аномалий)
