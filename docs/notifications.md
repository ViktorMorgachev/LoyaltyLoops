# Уведомления (пуш/почта) — концепт и API черновик

## Цели
- Партнёры могут рассылать акции своим клиентам (пуш, опционально e-mail).
- Платформа (супер-админ/супер-менеджер) рассылает системные уведомления.
- Клиент при тапе по пушу попадает на экран списка уведомлений в приложении.

## Роли и полномочия
- PLATFORM_SUPER_ADMIN, PLATFORM_SUPER_MANAGER — системные рассылки всем / выбранным сегментам (позже).
- PARTNER_ADMIN — рассылка только своим клиентам.
- PARTNER_MANAGER — опционально (пока нет).

## Модель уведомления (черновик)
```
Notification {
  id: string
  senderType: SYSTEM | PARTNER
  senderPartnerId?: string
  title: string
  body: string
  deepLink?: string   // для перехода в приложение (экран уведомлений)
  locale?: string     // язык контента (опционально)
  createdAt: number
}

UserNotification {
  id: string
  notificationId: string
  userId: string
  read: boolean
  deliveredAt: number
  readAt?: number
}
```

## API (черновик)
- POST `/notifications`  
  Создать уведомление и разослать. Body: `{ title, body, deepLink?, locale?, audience: "SYSTEM" | "PARTNER", partnerId? }`.  
  Доступ: SYSTEM (super admin/manager) или PARTNER_ADMIN для своего partnerId.
- GET `/notifications`  
  Список для текущего пользователя (мобильный клиент). Параметры: `limit/offset`, фильтр read.
- POST `/notifications/{id}/read`  
  Отметить прочитанным.

### E-mail (опционально)
- При создании уведомления можно передать `deliverEmail: true` — если у пользователя есть e-mail, отправить письмо (шаблон «Акция партнёра»).

## Клиент (мобильный) — поведение
- Экран «Мои уведомления»: список, бейдж «новое», возможность отметить прочитанным.
- При клике по пушу: открыть экран списка, отметить конкретное уведомление прочитанным.

## Пуш-доставка
- Использовать существующие FCM/APNs токены.
- Payload: `{ title, body, deepLink: "loyaltyloop://notifications" }`.

## Локализация
- Строки UI: список уведомлений, пустой список, «новое».
- Контент уведомлений — задаётся отправителем; поле `locale` хранится как подсказка, но не переводится автоматически.

## Ограничения и защита
- Rate-limit на создание уведомлений (особенно для PARTNER_ADMIN).
- Максимальная длина title/body.
- Аудит: логировать отправителя, аудиторию, количество получателей.
