# RFC: Мультивалютность и Глобальная Лояльность

## 1. Концепция (Overview)

Система переходит на модель **«Base Currency»**.
*   **Баланс клиента** всегда хранится в **Базовой валюте** Партнера (например, USD, если головной офис в США/Европе, или KGS, если это локальная сеть).
*   **Транзакции** происходят в **Валюте Точки** (Terminal Currency).
*   **Конвертация** происходит «на лету» в момент калькуляции и оплаты, используя закешированные курсы.

## 2. Изменения в Базе Данных (Exposed Tables)

### A. Таблица Партнеров (`PartnersTable`)
Добавляем настройки валюты хранения и политику начисления.

```kotlin
object PartnersTable : Table("partners") {
    // ... существующие поля ...

    // [NEW] Валюта, в которой физически хранится баланс в БД. 
    // ISO 4217 код: "USD", "KGS", "RUB", "KZT"
    val baseCurrency = varchar("base_currency", 3).default("USD")
}
```

### B. Таблица Курсов Валют (`ExchangeRatesTable`) — НОВАЯ
Справочник для конвертации. Обновляется фоновым джобом.

```kotlin
object ExchangeRatesTable : Table("exchange_rates") {
    // Составной ключ: ИЗ какой валюты -> В какую
    val fromCurrency = varchar("from_currency", 3) // Напр. "USD" (Base)
    val toCurrency = varchar("to_currency", 3)     // Напр. "KGS" (Terminal)
    
    // Множитель. Пример: 1 USD = 86.5 KGS. Rate = 86.5
    val rate = double("rate") 
    
    // Время последнего обновления (для мониторинга актуальности)
    val updatedAt = long("updated_at")

    override val primaryKey = PrimaryKey(fromCurrency, toCurrency)
}
```

### C. Таблица Транзакций (`TransactionsTable`)
Необходимо сохранять исторический курс, чтобы финансовые отчеты сходились.

```kotlin
object TransactionsTable : Table("transactions") {
    // ... существующие поля ...

    // [NEW] Валюта, в которой прошла оплата на кассе (например "KGS")
    val currency = varchar("currency", 3) 
    
    // [NEW] Курс на момент транзакции. 
    // Важно сохранить его, так как завтра курс в ExchangeRatesTable изменится.
    val exchangeRateSnapshot = double("exchange_rate_snapshot").default(1.0)
    
    // [NEW] Сколько бонусов реально списано/начислено в БАЗОВОЙ валюте (для отладки)
    val pointsBaseValue = double("points_base_value").default(0.0)
}
```

## 4. Алгоритм Расчета (Business Logic Flow)

### Сценарий:
*   **Партнер:** Base = `USD`.
*   **Точка:** Бишкек, Currency = `KGS`.
*   **Клиент:** Баланс = `10.0` (USD).
*   **Курс:** 1 USD = `85.0` KGS.
*   **Покупка:** Чек `5000` сом, Клиент хочет списать бонусы.

#### Шаг 1. Получение курса (Get Rate)
Сервер смотрит `TradingPoint.currency` (KGS) и `Partner.baseCurrency` (USD).
1.  Если равны -> Rate = 1.0.
2.  Если разные -> Ищем в `ExchangeRatesTable` пару USD -> KGS. Получаем `85.0`.

#### Шаг 2. Расчет Списания (Burn Calculation)
Максимально доступно для списания в местной валюте:
`MaxLocal = Card.Balance (10) * Rate (85) = 850 KGS`.

#### Шаг 3. Расчет Начисления (Earn Calculation)
Остаток к оплате: `5000 - 425 = 4575 KGS`.
Процент кэшбэка (из уровня карты): К примеру 10%.

1.  Начисление (Local): `4575 * 0.10 = 457.5 KGS`.
2.  Конвертация в базу (для сохранения): `457.5 / 85.0 = 5.38 USD` (округляем до 2 знаков).

#### Шаг 4. Коммит (Transaction Process)
При нажатии "Оплатить":
1.  `UPDATE cards SET balance = balance - 5.0 + 5.38` (Итого в базе стало 10.38 USD).
2.  `INSERT INTO transactions`:
    *   `amount`: 4575
    *   `currency`: "KGS"
    *   `exchangeRateSnapshot`: 85.0

## 5. Клиентское Приложение (UX & Timezones)

Что видит клиент в приложении? Это зависит не столько от Timezone, сколько от **геолокации** или выбранного **региона** магазина.

### A. Главная карточка лояльности
Тут мы показываем "честный" баланс и примерную конвертацию.

**Визуал:**
Если клиент в "Домашнем" регионе** (где валюта точки совпадает с валютой партнера, например в США)
> **LOYALTY CARD**
>
> **10.38Б (10.38$)** (Баланс в валюте системы)

**Визуал:**
Если клиент в "Роуминге"** (где валюта точки не совпадает с валютой партнера, например в KGS)
> **LOYALTY CARD**
>
> **10.38Б (882с)** (Баланс в валюте системы)

*Логика:* Приложение получает с бэкенда баланс в `baseCurrency` и список курсов, либо бэкенд сразу отдает массив `convertedBalances: [{currency: "KGS", amount: 882}, ...]`.
Баланс в UI отображаем в зависимости от региона пользователя вытаскивая convertedBalances данные

### B. Экран Филиалов (Map/List) (В будущем пока не трогаем)
Когда клиент открывает список филиалов, он видит разные страны.

1.  **Если клиент в "Домашнем" регионе** (где валюта точки совпадает с валютой партнера, например в США):
    *   На карте он видит: "10.38 Б (10.38$)"*.

2.  **Если клиент в "Роуминге"** (где валюта точки не совпадает с валютой партнера, например в KGS):
    *   Приложение определяет, что филиал работает в `KGS`.
    *   Использует курс с бэкенда.
    *   На карте он видит: "10.38 Б **(882с)**"*.

### Резюме по UX
*   **Таймзона:** Влияет только на отображение статуса "Открыто/Закрыто" и историю транзакций (время покупки показываем локальное).
*   **Валюта:** Определяется полем `currency` у конкретной торговой точки (`TradingPoint`).
*   **Кассир:** Всегда видит цифры в своей местной валюте. Он даже может не знать, что у партнера база в долларах. Система делает это прозрачно.
